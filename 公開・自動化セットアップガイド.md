## 1. GitHub でリポジトリを作成してファイルをアップロードする

GitHub は、ファイルをインターネット上に保存・管理するサービスです。
Netlify はこの GitHub のファイルを読み取ってサイトを公開します。

### 1-1. GitHub アカウントを作成する

1. ブラウザで **https://github.com** を開く
2. 右上の「**Sign up**」をクリック
3. 以下の情報を入力して登録する
   - メールアドレス
   - パスワード（8文字以上）
   - ユーザー名（例: `shunya-nakabayashi`）
4. 届いたメールに記載のコードを入力して認証完了

> ✅ すでに GitHub アカウントがある場合は、このステップをスキップしてください。

---

### 1-2. 新しいリポジトリ（フォルダ）を作成する

「リポジトリ」とは、GitHub 上のプロジェクト用フォルダのことです。

1. GitHub にログインした状態で、右上の「**+**」ボタンをクリック
2. 「**New repository**」を選択
3. 以下のように入力する

   | 項目 | 入力内容 |
   |---|---|
   | Repository name | `shunya-schedule` |
   | Description | （空欄でOK） |
   | Public / Private | **Public** を選択 |
   | Initialize this repository | **チェックを入れない**（重要） |

4. 「**Create repository**」ボタンをクリック

5. 作成後、以下のような画面が表示される。この画面は後で使うので **開いたままにしておく**

   ```
   …or push an existing repository from the command line
   git remote add origin https://github.com/あなたのユーザー名/shunya-schedule.git
   git branch -M main
   git push -u origin main
   ```

---

### 1-3. Git をパソコンにインストールする

Git は、ファイルをGitHub に送るためのツールです。

1. ブラウザで **https://git-scm.com/download/win** を開く
2. 「**Click here to download**」をクリックしてインストーラーをダウンロード
3. ダウンロードした `.exe` ファイルをダブルクリックして実行
4. インストール中の設定はすべて **「Next」のままでOK**（変更不要）
5. 「**Install**」→「**Finish**」でインストール完了

インストールできたか確認する：
1. キーボードの「**Windowsキー + R**」を押す
2. `cmd` と入力して Enter
3. 黒い画面（コマンドプロンプト）が開いたら以下を入力して Enter

   ```
   git --version
   ```

4. `git version 2.xx.x` のように表示されればOK

---

### 1-4. ファイルを GitHub にアップロードする

コマンドプロンプト（黒い画面）で作業します。

#### ① コマンドプロンプトを開く
1. キーボードの「**Windowsキー + R**」を押す
2. `cmd` と入力して Enter

#### ② サイトのフォルダに移動する
以下のコマンドをコピーして貼り付け（右クリック→貼り付け）、Enter を押す

```
cd C:\Work\ShunyaWebSite
```

#### ③ Git の初期設定（初回のみ）
名前とメールアドレスを登録します。`"` の中はご自身のものに変更してください。

```
git config --global user.name "Shunya Nakabayashi"
```
Enter を押してから：
```
git config --global user.email "あなたのメールアドレス@example.com"
```

#### ④ このフォルダを Git で管理し始める

```
git init
```

`Initialized empty Git repository` と表示されればOK

#### ⑤ すべてのファイルを「送る準備」に追加する

```
git add .
```

（何も表示されなければ成功）

#### ⑥ 「この状態を保存する」というメモを付ける

```
git commit -m "最初のアップロード"
```

`[main (root-commit) xxxxxxx] 最初のアップロード` のように表示されればOK

#### ⑦ GitHub のリポジトリと接続する

> ⚠️ 以下の `あなたのユーザー名` の部分を、**1-2 で作った GitHub のユーザー名**に変更してください

```
git remote add origin https://github.com/あなたのユーザー名/shunya-schedule.git
```

```
git branch -M main
```

#### ⑧ GitHub にファイルを送る（プッシュ）

```
git push -u origin main
```

ブラウザが開くか、ユーザー名とパスワードを聞かれる場合は、GitHub のものを入力してください。

**成功すると以下のように表示されます：**
```
Branch 'main' set up to track remote branch 'main' from 'origin'.
```

#### ⑨ GitHub で確認する
ブラウザで `https://github.com/あなたのユーザー名/shunya-schedule` を開き、
ファイルが一覧表示されていれば完了です！

---

> 💡 **今後ファイルを変更したとき**（スプレッドシート URL を config.js に設定した後など）は、
> 以下の 3 つのコマンドを順番に実行するだけで GitHub に反映されます：
>
> ```
> git add .
> git commit -m "更新内容のメモ"
> git push
> ```

## 2. Netlify でサイトを公開する

Netlify は GitHub のファイルを読み取って、無料でウェブサイトを公開してくれるサービスです。

1. ブラウザで **https://www.netlify.com** を開く
2. 「**Sign up**」→「**Sign up with GitHub**」でログイン（GitHub アカウントを使う）
3. ログイン後、「**Add new site**」→「**Import an existing project**」をクリック
4. 「**Deploy with GitHub**」を選択
5. `shunya-schedule` リポジトリを選択
6. 設定は何も変えず「**Deploy shunya-schedule**」ボタンをクリック
7. 数分後、`https://xxxxxxxx.netlify.app` のような URL が表示される

これが**あなたのサイトのURL**です！

> 💡 URL の名前は変更できます。「Site configuration」→「Change site name」から、
> たとえば `shunya-schedule.netlify.app` のように変更可能です。

## 3. Google スプレッドシートをデータ源として設定する

スプレッドシートを更新するだけでサイトのスケジュールが自動的に変わります。

### 3-1. スプレッドシートを作成する

1. **https://sheets.google.com** を開く（Google アカウントでログイン）
2. 「**＋ 新しいスプレッドシート**」を作成
3. シート名（左下のタブ）を「スケジュール」に変更
4. **1行目（ヘッダー行）** に以下を入力する

   | A | B | C | D | E | F | G |
   |---|---|---|---|---|---|---|
   | 日付 | 曜日 | 時間 | 会場 | 出演者 | チャージ | 備考 |

5. **2行目以降**にスケジュールを入力する。例：

   | A | B | C | D | E | F | G |
   |---|---|---|---|---|---|---|
   | 2026-04-11 | Sat | 19:30 | Pit Inn -新宿- | Shunya Nakabayashi (as) | ¥3,000 | ワンドリンク付き |

   > ⚠️ 日付は必ず `2026-04-11` のように **半角ハイフン区切り** で入力してください

### 3-2. スプレッドシートを公開する

1. メニューの「**ファイル**」→「**共有**」→「**ウェブに公開**」を選択
2. 「リンク」タブで以下を設定：
   - 左のドロップダウン：「**スケジュール**（シート名）」を選択
   - 右のドロップダウン：「**カンマ区切り値（.csv）**」を選択
3. 「**公開**」ボタンをクリック
4. 表示された URL（`https://docs.google.com/spreadsheets/d/e/xxxxx/pub?gid=0&single=true&output=csv` のような形式）を**コピーする**

### 3-3. config.js に URL を設定する

1. `C:\Work\ShunyaWebSite\config.js` をメモ帳などで開く
2. 以下の部分を見つける：

   ```js
   SHEETS_CSV_URL: "",
   ```

3. コピーした URL を `"` の中に貼り付ける：

   ```js
   SHEETS_CSV_URL: "https://docs.google.com/spreadsheets/d/e/xxxxx/pub?gid=0&single=true&output=csv",
   ```

4. ファイルを保存
5. コマンドプロンプトで以下を実行して GitHub に反映する：

   ```
   cd C:\Work\ShunyaWebSite
   git add .
   git commit -m "スプレッドシート連携を設定"
   git push
   ```

6. 数分後、Netlify が自動で更新を検知してサイトに反映されます

## 4. Make.com でメール自動化を設定する

「スケジュール追加」という件名のメールを送るだけで、自動的にスプレッドシートに追記されます。

### 4-1. Make.com アカウントを作成する

1. **https://www.make.com** を開く
2. 「**Sign up for free**」→ Google アカウントまたはメールで登録

### 4-2. シナリオを作成する

1. ログイン後、「**Create a new scenario**」をクリック
2. 「**+**」をクリックして最初のモジュールを追加
3. 「**Gmail**」を検索して選択 →「**Watch Emails**」を選択
4. Gmail アカウントを接続（「Add」→ Google アカウントでログイン）
5. 設定：
   - **Folder**: `INBOX`
   - **Criteria**: `Subject contains`「スケジュール追加」
6. 「**+**」をクリックして次のモジュールを追加
7. 「**Google Sheets**」を検索 →「**Add a Row**」を選択
8. 先ほど作ったスプレッドシートを選択し、列を以下のようにマッピングする

   > Make.com の「Text parser」でメール本文から各項目を取り出す高度な設定もできますが、
   > まずは以下の**シンプルなメール書式**で始めることをおすすめします

### 4-3. メールの送り方（日常の使い方）

以下の形式でメールを送るだけでスプレッドシートに追記されます：

```
宛先：あなた自身のGmailアドレス
件名：スケジュール追加

日付：2026-05-15
曜日：Fri
時間：19:30
会場：Pit Inn -新宿-
出演者：Shunya Nakabayashi (as), Taro Yamada (p)
チャージ：¥2,500
備考：
```

> 💡 スマートフォンのメールアプリからテンプレートを保存しておくと便利です。

## 5. note.com にリンクを追加する

サイトが公開されたら、note.com からリンクを追加しましょう。

1. note.com にログインし、マガジンのトップ記事を編集
2. 記事の最初または目立つ場所に以下を追加：

   ```
   ▶ 最新スケジュールはこちら
   https://shunya-schedule.netlify.app
   ```

3. 更新を保存

## 6. 日常の使い方（完成後）

セットアップが完了したら、日々の操作は以下のみです：

### ライブスケジュールを追加するとき

1. スマホまたはパソコンから、**自分のGmailに**メールを送る（上記の書式）
2. Make.com が自動でスプレッドシートに追記
3. 数分後、サイトに新しいスケジュールが表示される

### スケジュールを直接修正・削除するとき

1. **https://sheets.google.com** を開く
2. スプレッドシートで該当行を直接編集・削除
3. 保存すると自動的にサイトに反映される

## トラブルシューティング

| 症状 | 確認すること |
|---|---|
| サイトが更新されない | config.js の URL が正しいか確認 / スプレッドシートが「公開」されているか確認 |
| 日付の並びがおかしい | 日付が `2026-04-11` 形式（半角ハイフン）になっているか確認 |
| git push でエラー | GitHub のユーザー名・パスワードが正しいか確認 |
| Make.com が反応しない | シナリオが「ON」になっているか確認（トグルスイッチ） |
