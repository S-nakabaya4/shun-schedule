## 日常の使い方

### ▶ ライブ情報を追加する

1. スマホのホーム画面の「**フォーム**」アイコンをタップ
2. 以下の項目を入力して「**送信**」をタップ

   | 項目 | 入力例 | 補足 |
   |---|---|---|
   | 日付 | 2026/05/15 | カレンダーで選択 |
   | 開演時間 | 19:30 | 半角で入力 |
   | 会場 | Pit Inn -新宿- | |
   | 出演者 | Shunya Nakabayashi (as) | 複数行OK |
   | チャージ | ¥3,000 | |
   | 備考 | ワンドリンク付き | 任意・空欄OK |
   | URL | https://pit-inn.com/ | 任意・空欄なら会場名はリンクにならない |

3. 送信後、**数秒〜数分でサイトに反映**される

   > ✅ 曜日は日付から自動計算されます。入力不要です。

### ▶ ライブ情報を修正・削除する

1. [sheets.google.com](https://sheets.google.com) を開く（専用アカウントでログイン）
2. 「ライブスケジュール」→「**フォームの回答**」シートタブを開く
3. 該当行を直接編集・削除する
4. **自動でサイトに反映**される（保存ボタンは不要）

## サイトファイルを変更したときの反映手順

`index.html`・`style.css`・`config.js` などを編集した後は、
以下の 3 つのコマンドを**コマンドプロンプト**で実行してください。

```
cd C:\Work\ShunyaWebSite
git add .
git commit -m "変更内容のメモ"
git push
```

Netlify が自動で検知し、**1〜2 分後にサイトへ反映**されます。

> ⚠️ スプレッドシートのデータ更新はこの手順は不要です（自動反映）。
> この手順が必要なのは、**サイトのデザインや設定を変えたとき**だけです。

## スプレッドシートの公開設定を変更した場合

スプレッドシートを作り直したり、別のシートを使う場合は以下の手順で URL を更新します。

1. スプレッドシートで「**ファイル**」→「**共有**」→「**ウェブに公開**」
2. 対象シート（フォームの回答）を選択 → 「**カンマ区切り値（.csv）**」→「**公開**」
3. 表示された URL をコピー
4. `C:\Work\ShunyaWebSite\config.js` をメモ帳で開き、`SHEETS_CSV_URL` の `"..."` の中を書き換えて保存
5. コマンドプロンプトで push:
   ```
   cd C:\Work\ShunyaWebSite
   git add config.js
   git commit -m "スプレッドシートURL更新"
   git push
   ```

> ⚠️ URL の形式は必ず `…/pub?gid=…&output=csv` にしてください。
> `pubhtml` という形式は動作しません。

## トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| フォーム送信後もサイトに反映されない | スプレッドシートの「フォームの回答」シートに行が追加されているか確認。されていなければフォームとシートの連携を再確認。 |
| サイトにアクセスできない | Netlify のダッシュボードでデプロイが完了しているか確認。 |
| デザインや設定を変えたのに反映されない | `git add . / git commit / git push` を実行したか確認。`pubhtml` 形式の URL になっていないか確認。 |
| 日付の順番がおかしい | 日付列が `2026-04-11`（ハイフン区切り）または `2026/4/11`（スラッシュ）形式になっているか確認。 |
| スプレッドシートのデータが取得できない | 「ウェブに公開」の設定が残っているか確認（「Stop publishing」を押していないか）。 |
| 古い Netlify の URL を見ている | `69d1cc…--` で始まる URL は古いデプロイ。`visionary-entremet-e0424d.netlify.app` を使う。 |
