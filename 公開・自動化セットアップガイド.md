## 1. 専用 Google アカウントを作成する

スケジュール管理システム専用の Google アカウントを作成します。
個人アカウントと分けることで、管理がしやすくなります。

1. ブラウザで **https://accounts.google.com/signup** を開く
2. 以下を入力して登録する
   - 名前: `Shunya Schedule`（管理用の名前、何でもOK）
   - メールアドレス: `shunya.schedule.system@gmail.com`（例。好きな名前でOK）
   - パスワード
3. 電話番号認証を完了して登録終了

> ✅ このアカウントに、スプレッドシート・フォームをすべてまとめて保管します。

## 2. Google スプレッドシートと Google フォームを設定する

### 2-1. 専用アカウントでスプレッドシートを作成する

1. 専用アカウントでログインした状態で **https://sheets.google.com** を開く
2. 「**＋ 新しいスプレッドシート**」を作成
3. タイトルを「**ライブスケジュール**」に変更（左上のタイトル部分をクリック）
4. **1行目（ヘッダー行）** に以下を入力する（A1 から順番に）

   | A | B | C | D | E | F | G |
   |---|---|---|---|---|---|---|
   | 日付 | 曜日 | 時間 | 会場 | 出演者 | チャージ | 備考 |

### 2-2. スプレッドシートを「ウェブに公開」する

サイトがデータを読み取れるように公開設定をします。

1. メニューの「**ファイル**」→「**共有**」→「**ウェブに公開**」を選択
2. 左のドロップダウンを「**Sheet1**」に設定
3. 右のドロップダウンを「**カンマ区切り値（.csv）**」に設定
4. 「**公開**」ボタンをクリック → 確認ダイアログで「**OK**」
5. 表示された URL をコピーして **メモ帳などに一時保存** しておく

   > URL の例: `https://docs.google.com/spreadsheets/d/e/xxxxx/pub?gid=0&single=true&output=csv`

### 2-3. Google フォームを作成してスプレッドシートに接続する

Google フォームはスマホから入力できる「入力フォーム」です。
入力した内容が自動でスプレッドシートに追加されます。

#### フォームを作成する

1. **https://forms.google.com** を開く（専用アカウントでログイン済みの状態）
2. 「**＋ 新しいフォームを作成**」をクリック
3. タイトルを「**ライブスケジュール追加**」に変更

#### 質問を追加する（7項目）

「**＋**」ボタンで質問を追加しながら、以下の通りに設定します：

| 質問タイトル | 種類 | 設定 |
|---|---|---|
| 日付 | 日付 | 必須 ON |
| 曜日 | プルダウン | Mon/Tue/Wed/Thu/Fri/Sat/Sun を追加、必須 ON |
| 開演時間 | 記述式（短文） | 例: 19:30　必須 ON |
| 会場 | 記述式（短文） | 必須 ON |
| 出演者 | 段落（長文） | 必須 ON |
| チャージ | 記述式（短文） | 例: ¥2,500 |
| 備考 | 段落（長文） | （任意） |

> 「記述式」は質問右側の種類ドロップダウンから選べます。

#### フォームの回答をスプレッドシートに送る

1. フォーム編集画面の上部「**回答**」タブをクリック
2. 右上にある「**スプレッドシートにリンク**」アイコン（緑のシートアイコン）をクリック
3. 「**既存のスプレッドシートを選択する**」を選び、「**選択**」をクリック
4. 先ほど作った「**ライブスケジュール**」を選んで「**選択**」

   > ✅ これでフォームを送信するたびに、スプレッドシートに自動で行が追加されます。

#### フォームの URL をスマホに保存する

1. フォーム編集画面の右上「**送信**」ボタンをクリック
2. 「リンク」タブ（🔗 アイコン）を選択
3. 短縮 URL にチェックを入れて「**コピー**」
4. このリンクをスマホのホーム画面に追加する（下記参照）

   **iPhone の場合:**
   Safari でフォームを開く → 下の「共有」ボタン → 「ホーム画面に追加」

   **Android の場合:**
   Chrome でフォームを開く → 右上「⋮」→「ホーム画面に追加」

## 4. GitHub でリポジトリを作成してファイルをアップロードする

GitHub は、ファイルをインターネット上に保存・管理するサービスです。
Netlify はこの GitHub のファイルを読み取ってサイトを公開します。

> ⚠️ GitHub は**個人アカウント**で登録してください（専用アカウントとは別）。

### 4-1. GitHub アカウントを作成する

1. ブラウザで **https://github.com** を開く
2. 右上の「**Sign up**」をクリック
3. 以下の情報を入力して登録する
   - メールアドレス
   - パスワード（8文字以上）
   - ユーザー名（例: `shunya-nakabayashi`）
4. 届いたメールに記載のコードを入力して認証完了

> ✅ すでに GitHub アカウントがある場合は、このステップをスキップしてください。

---

### 4-2. 新しいリポジトリ（フォルダ）を作成する

「リポジトリ」とは、GitHub 上のプロジェクト用フォルダのことです。

1. GitHub にログインした状態で、右上の「**+**」ボタンをクリック
2. 「**New repository**」を選択
3. 以下のように入力する

   | 項目 | 入力内容 |
   |---|---|
   | Repository name | `shunya-schedule` |
   | Description | （空欄でOK） |
   | Public / Private | **Public** を選択 |
   | Initialize this repository | **チェックを入れない**（重要） |

4. 「**Create repository**」ボタンをクリック

5. 作成後、以下のような画面が表示される。この画面は後で使うので **開いたままにしておく**

   ```
   …or push an existing repository from the command line
   git remote add origin https://github.com/あなたのユーザー名/shunya-schedule.git
   git branch -M main
   git push -u origin main
   ```

---

### 4-3. Git をパソコンにインストールする

Git は、ファイルをGitHub に送るためのツールです。

1. ブラウザで **https://git-scm.com/download/win** を開く
2. 「**Click here to download**」をクリックしてインストーラーをダウンロード
3. ダウンロードした `.exe` ファイルをダブルクリックして実行
4. インストール中の設定はすべて **「Next」のままでOK**（変更不要）
5. 「**Install**」→「**Finish**」でインストール完了

インストールできたか確認する：
1. キーボードの「**Windowsキー + R**」を押す
2. `cmd` と入力して Enter
3. 黒い画面（コマンドプロンプト）が開いたら以下を入力して Enter

   ```
   git --version
   ```

4. `git version 2.xx.x` のように表示されればOK

### 4-4. ファイルを GitHub にアップロードする

コマンドプロンプト（黒い画面）で作業します。

#### ① コマンドプロンプトを開く
1. キーボードの「**Windowsキー + R**」を押す
2. `cmd` と入力して Enter

#### ② サイトのフォルダに移動する
以下のコマンドをコピーして貼り付け（右クリック→貼り付け）、Enter を押す

```
cd C:\Work\ShunyaWebSite
```

#### ③ Git の初期設定（初回のみ）
名前とメールアドレスを登録します。`"` の中はご自身のものに変更してください。

```
git config --global user.name "Shunya Nakabayashi"
```
Enter を押してから：
```
git config --global user.email "あなたのメールアドレス@example.com"
```

#### ④ このフォルダを Git で管理し始める

```
git init
```

`Initialized empty Git repository` と表示されればOK

#### ⑤ すべてのファイルを「送る準備」に追加する

```
git add .
```

（何も表示されなければ成功）

#### ⑥ 「この状態を保存する」というメモを付ける

```
git commit -m "最初のアップロード"
```

`[main (root-commit) xxxxxxx] 最初のアップロード` のように表示されればOK

#### ⑦ GitHub のリポジトリと接続する

> ⚠️ 以下の `あなたのユーザー名` の部分を、**1-2 で作った GitHub のユーザー名**に変更してください

```
git remote add origin https://github.com/あなたのユーザー名/shunya-schedule.git
```

```
git branch -M main
```

#### ⑧ GitHub にファイルを送る（プッシュ）

```
git push -u origin main
```

ブラウザが開くか、ユーザー名とパスワードを聞かれる場合は、GitHub のものを入力してください。

**成功すると以下のように表示されます：**
```
Branch 'main' set up to track remote branch 'main' from 'origin'.
```

#### ⑨ GitHub で確認する
ブラウザで `https://github.com/あなたのユーザー名/shunya-schedule` を開き、
ファイルが一覧表示されていれば完了です！

---

> 💡 **今後ファイルを変更したとき**（スプレッドシート URL を config.js に設定した後など）は、
> 以下の 3 つのコマンドを順番に実行するだけで GitHub に反映されます：
>
> ```
> git add .
> git commit -m "更新内容のメモ"
> git push
> ```

## 5. Netlify でサイトを公開する

Netlify は GitHub のファイルを読み取って、無料でウェブサイトを公開してくれるサービスです。

1. ブラウザで **https://www.netlify.com** を開く
2. 「**Sign up**」→「**Sign up with GitHub**」でログイン（GitHub アカウントを使う）
3. ログイン後、「**Add new site**」→「**Import an existing project**」をクリック
4. 「**Deploy with GitHub**」を選択
5. `shunya-schedule` リポジトリを選択
6. 設定は何も変えず「**Deploy shunya-schedule**」ボタンをクリック
7. 数分後、`https://xxxxxxxx.netlify.app` のような URL が表示される

これが**あなたのサイトのURL**です！

> 💡 URL の名前は変更できます。「Site configuration」→「Change site name」から、
> たとえば `shunya-schedule.netlify.app` のように変更可能です。

サイトが公開されたら、note.com からリンクを追加しましょう。

1. note.com にログインし、マガジンのトップ記事を編集
2. 記事の最初または目立つ場所に以下を追加：

   ```

3. 更新を保存

## 7. 日常の使い方（完成後）

セットアップが完了したら、日々の操作は以下のみです：

### ライブスケジュールを追加するとき

1. スマホのホーム画面に保存したフォームのアイコンをタップ
2. 日付・会場・出演者などを入力して「**送信**」をタップ
3. **すぐに**サイトへ反映される（数秒〜数分）

### スケジュールを修正・削除するとき

1. スマホまたはパソコンで **https://sheets.google.com** を開く（専用アカウントでログイン）
2. 「ライブスケジュール」を開いて該当行を直接編集・削除
3. 保存するとサイトに自動反映される

## トラブルシューティング

| 症状 | 確認すること |
|---|---|
| サイトが更新されない | config.js の URL が正しいか確認 / スプレッドシートが「公開」されているか確認 |
| 日付の並びがおかしい | 日付が `2026-04-11` 形式（半角ハイフン）になっているか確認 |
| git push でエラー | GitHub のユーザー名・パスワードが正しいか確認 |
| フォームを送信してもサイトに反映されない | スプレッドシートのフォーム回答タブに行が追加されているか確認 |
